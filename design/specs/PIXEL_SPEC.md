# Especificação dimensional para reprodução

A referência foi normalizada para uma tela mobile estreita em proporção aproximada 9:21.7. O código deve usar proporções responsivas, mas preservar estas relações:

## Início

- padding lateral: 16 dp;
- cabeçalho: 62–68 dp;
- cartão de resumo: 250–280 dp, raio 22–26 dp;
- CTA: 48–54 dp;
- cartão de propriedade: 84–92 dp;
- navegação inferior: 68–76 dp;
- distância entre cartões: 10–12 dp.

## Mapa

- cabeçalho: 66–72 dp;
- chips: 40–44 dp;
- controles flutuantes: 46–50 dp;
- sheet recolhida: 34–38% da altura útil;
- raio superior do sheet: 26–30 dp;
- stroke principal do polígono: 3–4 dp;
- vértices: 12–15 dp.

## Passaporte

- header: 62–68 dp;
- cartão de identidade: 120–138 dp;
- gaps entre cartões: 10–12 dp;
- métrica principal: 26–30 sp;
- progress nodes: 18–22 dp.

## Neon

O núcleo deve ser uma linha sólida. Halo é uma segunda camada visual com baixa opacidade. Não substitua a linha por blur. No claro, halo quase invisível; no escuro, até 12 px equivalentes.
